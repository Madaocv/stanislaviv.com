Documentation¶

Before starting to work with template, there are some basic concepts/technologies you need to be familiar with. Please check the Bootstrap framework and SASS for more information. These two technologies and the core of the template.

There are some other essentials technologies used across the template files. Font Awesome is used for displaying the icons or jQuery for basic DOM manipulation. However these technogies are not mandatory to use. But learning these technologies will make easier site creation.

File Structure¶

All CSS files are located inside assets/css folder. Template oriented files are generated from SASS source codes. You can find the SASS files insidede assets/scss. It is recommended to edit SASS files instead of CSS files because if you compile the SASS, you will loose all your changes.

Images are located in assets/img. The tmp folder container sample images. In the package which you downloaded are situated the compressed images with the placeholders only.

Folder Description
assets/css CSS files
assets/data JSON data for eg. maps
assets/fonts Font families
assets/img Images
assets/libraries Libraries
assets/scss SASS files

JavaScript¶

The main JavaScript files are located insided assets/js folder. The file is always named by template name.

Each JavaScript functionality has its own code block. So if you are not interested in gallery functionality you can just remove that block and save the file.

Custom Colors¶

You can change the colors by editing and compiling the SASS file. File which contains all colors is located inside assets/scss/helpers/_variables.scss. You can change the primary color which is stored in $color-primary variable.

After successful changing the primary color you can compile the SASS files. You must run the gulp compile command inside the root template directory. All CSS files will be replaced with the new version.

1
2
$color-primary: #0BB7A5; // Change this variable
$color-secondary: #4E4A4A;

$ cd template-name
$ npm install gulp gulp-sass
$ gulp compile

Open Street Maps & Google Maps¶

The data for the maps are coming from the assets/data/markers.json file. The data are downloaded from the JSON file via the AJAX call. If you want to reuse this file for your custom markers, please use the same data structure and do not remove any JSON properties.

var access_token = 'YOUR_ACCESS_TOKEN_HERE';

L.tileLayer('https://api.tiles.mapbox.com/v4/{id}/{z}/{x}/{y}.png?access_token=' + access_token, {
    scrollWheelZoom: false,
    id: 'mapbox.streets',
    attribution: '<a href="http://www.mapbox.com/about/maps/" target="_blank">Terms &amp; Feedback</a>'
}).addTo(map);

When using the leaflet please open main JavaScript file which is located inside assets/js folder. Open that file and lookg for the access_token variable which you will replace by your own token. You can create your own access token at www.mapbox.com.
